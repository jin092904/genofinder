### Cohort design v2 — sample factor 분포를 prompt 에 포함 (2026-05-12 같은 날 추가)

**문제 발견**: GSE317412 (mouse aging study) live 결과 v1 이 "control vs treatment" 로 잘못 라벨링. 실제 그룹 변수는 `age` (12-weeks vs 68-weeks). LLM 이 abstract 만으로는 그룹 구조를 못 잡고 추측. samples 의 raw_attributes 분포가 prompt 에 없었음.

**개선**:
- [x] **`fetch_sample_factors`** (api `cohort_samples` + workers `indexer.samples` 양쪽) — raw_attributes 의 모든 key 를 sample 다양성으로 분류. `varying` (값 종류 ≥ 2 → 그룹 변수 후보) / `constant` (모두 동일 → context). administrative 키 (Sample_status / Sample_contact_* 등) 38개는 `_FACTOR_EXCLUDE_KEYS` 로 제외.
- [x] **prompt 확장** — `SAMPLE FACTORS` 블록 추가. Group variables 와 Constant variables 분리 출력. system prompt 에 "treat each varying factor value as a group with n=its sample count" 명시.
- [x] **`cohort-v2-phi4-2026-05-12`** 로 version bump. workers / api 양쪽 sync.
- [x] **`POST /cohort/extract`** 가 `fetch_sample_factors` 호출 후 extractor 에 전달.
- [x] **테스트 +5** (api 4 + workers 1) — varying / constant / cap / 빈 입력 / v2 version. workers 53 + api 66 모두 통과.
- [x] **Live retry** (GSE317412 force=true, 44s):
  - v1: `control` (case) + `treatment` (other), design_type=unknown
  - **v2: `young` (case, "aged at 12-weeks") + `old` (comparison, "aged at 68-weeks"), design_type=cohort, notes="The study compares aged hearts with a time component."**

- **Live 검증**: `POST /search {"query_text":"GSE317412"}` → **#1 GEO GSE317412** (rerank score top, lexical 143.26).
